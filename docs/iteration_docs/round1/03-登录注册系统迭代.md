## 迭代概述

### 背景

当前登录注册系统存在严重问题：
- 每次进入应用都显示登录页（Token 持久化失败）
- 点击按钮没有正确跳转（路由混乱）
- 缺少邮箱验证码功能
- 缺少密码重置功能
- 登录流程不够健壮

### 目标

打造一个**正常、常规、健壮**的登录注册系统：
- 修复 Token 持久化问题
- 优化登录流程，确保状态正确保持
- 添加邮箱验证码功能（仅注册需要）
- 添加密码重置功能
- 提升用户体验和系统稳定性

### 核心改进点

| 改进点 | 当前状态 | 目标状态 |
|--------|----------|----------|
| Token 持久化 | ❌ 经常失败 | ✅ 稳定可靠 |
| 登录流程 | ❌ 每次需登录 | ✅ 自动登录 |
| 邮箱验证 | ❌ 无 | ✅ 注册需要 |
| 密码重置 | ❌ 无 | ✅ 完整功能 |
| 错误处理 | ❌ 不足 | ✅ 完善友好 |

## 问题分析

### 问题 1：每次进入应用都显示登录页

**现象**：
- 无论上次是否登录，重新打开应用后都需要登录
- Token 似乎没有保存

**根本原因**：

1. **Token 保存失败** (`mobile/src/services/storage/tokenStorage.ts`)
   - 文件系统操作可能失败，但没有错误处理
   - 没有重试机制
   - 写入失败时静默失败

2. **checkAuth() 超时逻辑错误** (`mobile/src/stores/authStore.ts`)
   ```typescript
   const timeout = new Promise<never>((_, reject) => {
     setTimeout(() => reject(new Error('Auth check timeout')), 5000);
   });
   ```
   - 5 秒超时太短
   - 首次加载时文件系统慢，容易超时
   - 超时后强制结束，导致状态混乱

3. **touchTokenSavedAt() 逻辑问题**
   - 如果 `tokenSavedAt === null`，会更新为当前时间
   - 这可能导致旧 Token 被错误激活

### 问题 2：路由跳转不正确

**现象**：
- 点击"开始使用"、"邮箱登录"、"注册新账号"按钮时，没有跳转到对应页面
- 用户感觉"都是直接进去了"

**根本原因**：

1. **"开始使用"按钮行为** (`mobile/app/(auth)/index.tsx`)
   ```typescript
   const success = await register(nickname || undefined);
   if (success) {
     router.replace('/(tabs)'); // 直接跳转到主界面
   }
   ```
   - 用户期望跳转到登录/注册页面
   - 实际上直接进入主界面（设备ID注册）

2. **路由配置问题**
   - `mobile/app/_layout.tsx` 中的路由逻辑可能有问题
   - 需要检查 `isAuthenticated` 状态是否正确

### 问题 3：缺少邮箱验证码功能

**当前状态**：
- 注册流程：邮箱 + 密码 → 直接注册
- 登录流程：邮箱 + 密码 → 直接登录

**风险**：
- 无法验证邮箱是否真实有效
- 可能被恶意注册
- 无法防止垃圾账号

### 问题 4：缺少密码重置功能

**当前状态**：
- 登录页有"忘记密码"链接，但点击无反应
- 用户忘记密码后无法找回

**风险**：
- 用户体验差
- 用户可能因此放弃使用

#### 1.1 添加重试机制

**文件**: `mobile/src/services/storage/tokenStorage.ts`

```typescript
async function writeNativeKvFile(kv: NativeKv): Promise<void> {
  const fs = await getFileSystem();
  const { path } = await getNativePaths();

  let retries = 3;
  let lastError: unknown = null;

  while (retries > 0) {
    try {
      await ensureNativeDir();
      const data = JSON.stringify(kv);
      await fs.writeAsStringAsync(path, data);
      console.log('[TokenStorage] Successfully saved to:', path);
      return;
    } catch (e) {
      console.warn(`[TokenStorage] Write failed, retries left: ${retries - 1}`, e);
      lastError = e;
      retries--;
      if (retries > 0) {
        await new Promise(r => setTimeout(r, 100)); // 等待100ms重试
      }
    }
  }

  throw new Error(`[TokenStorage] Failed to save after retries: ${lastError}`);
}

// 修改 queueNativeWrite 调用新的写入函数
async function queueNativeWrite(): Promise<void> {
  nativeKvWriteChain = nativeKvWriteChain
    .catch(() => undefined)
    .then(async () => {
      if (nativeKvCache) {
        try {
          await writeNativeKvFile(nativeKvCache);
        } catch (e) {
          console.error('[TokenStorage] Failed to persist token:', e);
          throw e; // 重新抛出错误，让上层感知
        }
      }
    });
  return nativeKvWriteChain;
}
```

```typescript
checkAuth: async () => {
  set({ isLoading: true });

  try {
    // 移除超时保护，让请求自然完成
    const [info, token, tokenSavedAt] = await Promise.all([
      getLocalUserInfo(),
      tokenStorage.getToken(),
      tokenStorage.getTokenSavedAt(),
    ]);

    // 验证 Token 存在且未过期
    if (token) {
      const now = Date.now();
      const tokenAge = tokenSavedAt ? (now - tokenSavedAt) : 0;
      const isExpired = tokenAge > TOKEN_TTL_MS;

      if (isExpired) {
        console.log('[Auth] Token expired, clearing...');
        await tokenStorage.clearAll();
        set({
          token: null,
          userId: null,
          deviceId: null,
          email: null,
          isAuthenticated: false,
          isLoading: false,
        });
        return;
      }

      // Token 有效
      console.log('[Auth] Token valid, user authenticated');
      set({
        token,
        userId: info.userId,
        deviceId: info.deviceId,
        email: info.email,
        isAuthenticated: true,
        isLoading: false,
      });
      return;
    }

    // 没有 Token
    console.log('[Auth] No token found');
    set({
      token: null,
      userId: null,
      deviceId: null,
      email: null,
      isAuthenticated: false,
      isLoading: false,
    });
  } catch (error) {
    console.error('[Auth] Check auth failed:', error);
    set({
      token: null,
      userId: null,
      deviceId: null,
      email: null,
      isAuthenticated: false,
      isLoading: false,
    });
  }
}
```

```python
@router.post("/send-verification-code", response_model=ApiResponse[dict])
def send_verification_code(
    payload: EmailCodeRequest,
    db: Session = Depends(get_db),
) -> ApiResponse[dict]:
    """发送邮箱验证码"""
    # 检查邮箱是否已注册
    stmt = select(User).where(User.email == payload.email.lower())
    existing_user = db.scalar(stmt)

    if payload.purpose == "register" and existing_user:
        raise HTTPException(
            status_code=status.HTTP_409_CONFLICT,
            detail={"code": EMAIL_ALREADY_EXISTS, "message": "邮箱已被注册"},
        )

    # 生成 6 位验证码
    code = ''.join([str(random.randint(0, 9)) for _ in range(6)])
    expires_at = datetime.now() + timedelta(minutes=10)

    # 保存验证码
    if existing_user:
        existing_user.verification_code = code
        existing_user.verification_expires_at = expires_at
    else:
        temp_user = User(
            email=payload.email.lower(),
            verification_code=code,
            verification_expires_at=expires_at,
        )
        db.add(temp_user)

    db.commit()

    # 发送邮件
    await send_verification_email(payload.email, code)

    return ApiResponse.ok({"message": "验证码已发送"})
```

```python
    stmt = select(User).where(
```

```python
def reset_password(

    """重置密码"""
    stmt = select(User).where(User.email == payload.email.lower())
```

```typescript
  async (error) => {

      return Promise.reject(new Error('网络连接失败，请检查网络设置'));
    }

    if (error.response?.status === 401) {

      await tokenStorage.clearAll();

      return Promise.reject(new Error('登录已过期，请重新登录'));

    if (error.response?.status === 500) {
      return Promise.reject(new Error('服务器错误，请稍后重试'));
    }

    if (error.code === 'ECONNABORTED') {
      return Promise.reject(new Error('请求超时，请检查网络'));
    }

    return Promise.reject(error);
  }
);
```

#### 5.2 请求重试机制

**改动**:
```typescript
export async function fetchWithRetry<T>(
  fn: () => Promise<T>,
  maxRetries: number = 2,
  delay: number = 1000
): Promise<T> {
  let lastError: unknown;

  for (let i = 0; i <= maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error;
      if (i < maxRetries) {
        console.warn(`Request failed, retry ${i + 1}/${maxRetries}`);
        await new Promise(r => setTimeout(r, delay * (i + 1)));
      }
    }
  }

  throw lastError;
}
```

```typescript
interface VerificationCodeInputProps {
  length?: number;
  value: string;
  onChange: (code: string) => void;
  onSendCode: () => void;
  canSendCode: boolean;
  countdown: number;
}

export function VerificationCodeInput({
  length = 6,
  value,
  onChange,
  onSendCode,
  canSendCode,
  countdown,
}: VerificationCodeInputProps) {
  const inputRefs = useRef<Array<TextInput | null>>([]);

  const handleChange = (text: string, index: number) => {
    const newValue = value.split('');
    newValue[index] = text;
    const finalValue = newValue.join('').slice(0, length);

    onChange(finalValue);

    // 自动聚焦下一个输入框
    if (text && index < length - 1) {
      inputRefs.current[index + 1]?.focus();
    }
  };

  const handleKeyPress = (key: string, index: number) => {
    // 删除时聚焦上一个输入框
    if (key === 'Backspace' && !value[index] && index > 0) {
      inputRefs.current[index - 1]?.focus();
    }
  };

  return (
    <View style={styles.container}>
      <View style={styles.inputsContainer}>
        {Array.from({ length }).map((_, index) => (
          <TextInput
            key={index}
            ref={(ref) => (inputRefs.current[index] = ref)}
            style={styles.input}
            value={value[index] || ''}
            onChangeText={(text) => handleChange(text, index)}
            onKeyPress={({ nativeEvent }) => handleKeyPress(nativeEvent.key, index)}
            keyboardType="number-pad"
            maxLength={1}
            textAlign="center"
          />
        ))}
      </View>

      <Pressable
        style={[
          styles.sendButton,
          !canSendCode && styles.sendButtonDisabled,
        ]}
        onPress={onSendCode}
        disabled={!canSendCode}
      >
        <Text style={styles.sendButtonText}>
          {countdown > 0
            ? `重新发送(${countdown}s)`
            : '发送验证码'}
        </Text>
      </Pressable>
    </View>
  );
}
```

#### 密码强度指示器

```typescript
interface PasswordStrengthProps {
  password: string;
}

function calculatePasswordStrength(password: string): {
  score: number;
  label: string;
  color: string;
} {
  let score = 0;

  if (password.length >= 8) score += 1;
  if (password.length >= 12) score += 1;
  if (/[a-z]/.test(password)) score += 1;
  if (/[A-Z]/.test(password)) score += 1;
  if (/[0-9]/.test(password)) score += 1;
  if (/[^a-zA-Z0-9]/.test(password)) score += 1;

  const levels = [
    { max: 2, label: '弱', color: '#EF4444' },
    { max: 4, label: '中等', color: '#F59E0B' },
    { max: 6, label: '强', color: '#10B981' },
  ];

  const level = levels.find(l => score <= l.max) || levels[levels.length - 1];
  return { score, ...level };
}

export function PasswordStrength({ password }: PasswordStrengthProps) {
  const { score, label, color } = calculatePasswordStrength(password);

  return (
    <View style={styles.container}>
      <View style={styles.barsContainer}>
        {[0, 1, 2, 3].map((i) => (
          <View
            key={i}
            style={[
              styles.bar,
              { backgroundColor: i < score ? color : '#E5E7EB' },
            ]}
          />
        ))}
      </View>
      <Text style={[styles.label, { color }]}>
        {label}
      </Text>
    </View>
  );
}
```

## 风险评估

### 技术风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| Token 持久化在部分设备上失败 | 中 | 高 | 添加多重存储策略和错误处理 |
| 邮件服务集成复杂 | 高 | 中 | 使用成熟的 SMTP 库，提供 fallback 方案 |
| 网络检测不准确 | 低 | 中 | 结合多种检测方式 |
| 验证码被滥用 | 中 | 中 | 添加频率限制和图形验证码 |
