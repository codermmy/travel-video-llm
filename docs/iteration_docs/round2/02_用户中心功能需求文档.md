# 用户中心功能需求文档

> **文档版本**: v1.0
> **创建时间**: 2026-02-10
> **目标**: 完善登录/注册功能后的用户中心，实现用户信息展示和管理

---

## 📋 需求概述

### 当前状态
1. **认证功能已实现**: 设备ID注册、邮箱注册、登录、邮箱验证、密码重置
2. **基础用户模型**: User 模型包含 id、device_id、email、nickname 等字段
3. **缺少用户中心**: 当前 tabs 只有"足迹"和"事件"两个页面

### 核心目标
- 在底部 tabs 中添加"用户中心"（我的）页面
- 展示用户信息：用户名称、用户邮箱、用户头像
- 实现用户管理功能：用户重命名
- 后端扩展：支持通过用户ID、用户名称、用户昵称查询

## 🎯 核心功能需求

### 1. 用户中心页面

**功能描述**
- 在底部 tabs 中添加第三个标签页："用户中心"（我的）
- 点击进入后展示用户个人信息和管理入口

**导航结构**
```
底部 Tabs：
  - 足迹（地图）- 已有
  - 事件 - 已有
  - 用户中心（我的）- 新增
```

---

### 2. 用户信息展示

**功能描述**
- 显示当前登录用户的基本信息
- 支持快速编辑

**展示字段**

| 字段 | 说明 | 来源 |
|------|------|------|
| 用户头像 | 圆形头像，点击可上传/更换 | User.avatar_url（新增字段） |
| 用户昵称 | 用户设置的昵称 | User.nickname |
| 用户邮箱 | 用户注册的邮箱 | User.email |
| 用户ID | 唯一标识符 | User.id |
| 注册时间 | 账户创建时间 | User.created_at |
| 认证方式 | 设备ID 或 邮箱 | User.auth_type |

**UI 布局**
- 顶部：圆形头像（120x120），居中显示
- 头像下方：用户昵称（大标题）
- 昵称下方：用户邮箱（副标题）
- 底部：功能列表（编辑资料、设置、退出登录）

---

### 3. 用户重命名

**功能描述**
- 用户可以修改自己的昵称
- 昵称限制：2-64 个字符

**交互流程**
1. 在用户中心点击"编辑资料"
2. 进入编辑页面，显示当前昵称
3. 修改昵称后保存
4. 返回用户中心，显示更新后的昵称

**约束**
- 昵称不能为空
- 昵称长度限制：2-64 个字符
- 允许的字符：中文、英文、数字、下划线、连字符

---

### 4. 头像上传

**功能描述**
- 用户可以上传自定义头像
- 支持从相册选择或拍照
- 图片限制：1MB，支持格式：JPEG、PNG

**交互流程**
1. 在用户中心点击头像
2. 选择图片来源（相册/拍照）
3. 裁剪/预览头像
4. 上传到服务器
5. 更新用户头像 URL

**头像尺寸规范**
- 原始上传：限制 1MB
- 存储尺寸：正方形，200x200 或 400x400（压缩）
- 展示尺寸：120x120（用户中心）

**默认头像**
- 用户未设置头像时，显示默认头像
- 默认头像样式：渐变色背景 + 首字母

---

### 5. 后端用户查询扩展

**功能描述**
- 支持通过多种标识符查询用户信息
- 用于搜索、社交分享、用户互动等功能

**查询方式**

| 查询方式 | API 端点 | 参数 | 说明 |
|---------|---------|------|------|
| 用户ID | `GET /api/v1/users/:user_id` | user_id | 唯一标识符，精确查询 |
| 用户名称 | `GET /api/v1/users/by-username/:username` | username | 用户自定义名称，精确查询 |
| 用户昵称 | `GET /api/v1/users/by-nickname/:nickname` | nickname | 用户显示名称，模糊查询 |
| 当前用户 | `GET /api/v1/users/me` | - | 获取当前登录用户信息 |

**搜索结果格式**

**2. 更新当前用户信息**

```python
class UserUpdateRequest(BaseModel):
    nickname: Optional[str] = Field(None, min_length=2, max_length=64)
    avatar_url: Optional[str] = Field(None, max_length=512)

@router.patch("/me", response_model=ApiResponse[UserResponse])
def update_current_user(
    payload: UserUpdateRequest,
    current_user_id: CurrentUserIdDep,
    db: Session = Depends(get_db)
) -> ApiResponse[UserResponse]:
    """更新当前用户信息"""
    user = db.scalar(select(User).where(User.id == current_user_id))

    if not user:
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail="User not found")

    # 更新昵称
    if payload.nickname is not None:
        user.nickname = payload.nickname

    # 更新头像
    if payload.avatar_url is not None:
        user.avatar_url = payload.avatar_url

    db.commit()
    db.refresh(user)

    return ApiResponse.ok(_user_to_response(user))
```

**5. 通过用户昵称搜索**

```python
class UserSearchResponse(BaseModel):
    users: List[UserResponse]
    total: int

@router.get("/by-nickname/{nickname}", response_model=ApiResponse[UserSearchResponse])
def search_users_by_nickname(
    nickname: str,
    page: int = 1,
    page_size: int = 20,
    db: Session = Depends(get_db)
) -> ApiResponse[UserSearchResponse]:
    """通过用户昵称模糊搜索用户"""
    query = select(User).where(User.nickname.ilike(f"%{nickname}%"))

    # 分页
    total = db.scalar(func.count(query))
    results = db.scalars(
        query.offset((page - 1) * page_size).limit(page_size)
    ).all()

    users = [_user_to_response(user) for user in results]

    return ApiResponse.ok(UserSearchResponse(users=users, total=total))
```

```typescript
import React, { useState } from 'react';
import { View, Text, TextInput, TouchableOpacity, Alert, ActivityIndicator } from 'react-native';
import { useRouter, useLocalSearchParams } from 'expo-router';
import { userApi } from '@/services/api/userApi';
import { authStorage } from '@/utils/authStorage';

export default function EditProfileScreen() {
  const router = useRouter();
  const { nickname: initialNickname } = useLocalSearchParams<{ nickname?: string }>();
  const [nickname, setNickname] = useState(initialNickname || '');
  const [saving, setSaving] = useState(false);

  const handleSave = async () => {
    // 验证
    if (!nickname.trim()) {
      Alert.alert('提示', '昵称不能为空');
      return;
    }

    if (nickname.length < 2 || nickname.length > 64) {
      Alert.alert('提示', '昵称长度必须在 2-64 个字符之间');
      return;
    }

    setSaving(true);

    try {
      await userApi.updateCurrentUser({ nickname: nickname.trim() });
      Alert.alert('成功', '昵称已更新', [
        { text: '确定', onPress: () => router.back() },
      ]);
    } catch (error) {
      Alert.alert('失败', '更新昵称失败，请重试');
    } finally {
      setSaving(false);
    }
  };

  return (
    <View style={styles.container}>
      <View style={styles.header}>
        <Text style={styles.title}>编辑昵称</Text>
      </View>

      <View style={styles.form}>
        <Text style={styles.label}>昵称</Text>
        <TextInput
          style={styles.input}
          value={nickname}
          onChangeText={setNickname}
          placeholder="请输入昵称"
          maxLength={64}
          autoFocus
        />
        <Text style={styles.hint}>
          昵称长度：{nickname.length}/64
        </Text>
      </View>

      <TouchableOpacity
        style={[styles.button, saving && styles.buttonDisabled]}
        onPress={handleSave}
        disabled={saving}
      >
        {saving ? (
          <ActivityIndicator color="#fff" />
        ) : (
          <Text style={styles.buttonText}>保存</Text>
        )}
      </TouchableOpacity>
    </View>
  );
}
```

```typescript
import React, { useState, useRef } from 'react';
import { View, Text, TouchableOpacity, Alert, Image, ActivityIndicator } from 'react-native';
import { useRouter } from 'expo-router';
import * as ImagePicker from 'expo-image-picker';
import { userApi } from '@/services/api/userApi';
import { uploadPhotoToOss } from '@/services/uploadService';

export default function AvatarUploadScreen() {
  const router = useRouter();
  const [avatar, setAvatar] = useState<string | null>(null);
  const [uploading, setUploading] = useState(false);

  const pickAvatar = async () => {
    try {
      const result = await ImagePicker.launchImageLibraryAsync({
        mediaTypes: ImagePicker.MediaTypeOptions.Images,
        allowsEditing: true,
        aspect: [1, 1],
        quality: 0.8,
      });

      if (!result.canceled && result.assets[0]) {
        setAvatar(result.assets[0].uri);
      }
    } catch (error) {
      Alert.alert('错误', '选择图片失败');
    }
  };

  const handleUpload = async () => {
    if (!avatar) return;

    setUploading(true);

    try {
      // 上传到 OSS
      const avatarUrl = await uploadPhotoToOss(avatar, 'avatars');

      // 更新用户头像
      await userApi.updateCurrentUser({ avatar_url: avatarUrl });

      Alert.alert('成功', '头像已更新', [
        { text: '确定', onPress: () => router.back() },
      ]);
    } catch (error) {
      Alert.alert('失败', '上传头像失败，请重试');
    } finally {
      setUploading(false);
    }
  };

  return (
    <View style={styles.container}>
      <View style={styles.header}>
        <TouchableOpacity onPress={router.back}>
          <Ionicons name="arrow-back" size={24} color="#333" />
        </TouchableOpacity>
        <Text style={styles.title}>上传头像</Text>
      </View>

      <View style={styles.preview}>
        {avatar ? (
          <Image source={{ uri: avatar }} style={styles.imagePreview} />
        ) : (
          <View style={styles.placeholder}>
            <Ionicons name="image-outline" size={64} color="#ccc" />
            <Text style={styles.placeholderText}>选择头像</Text>
          </View>
        )}
      </View>

      <View style={styles.actions}>
        <TouchableOpacity
          style={[styles.button, !avatar && styles.buttonDisabled]}
          onPress={pickAvatar}
          disabled={uploading}
        >
          <Text style={styles.buttonText}>
            {avatar ? '重新选择' : '选择图片'}
          </Text>
        </TouchableOpacity>

        <TouchableOpacity
          style={[styles.button, styles.primaryButton, (!avatar || uploading) && styles.buttonDisabled]}
          onPress={handleUpload}
          disabled={!avatar || uploading}
        >
          {uploading ? (
            <ActivityIndicator color="#fff" />
          ) : (
            <Text style={styles.buttonText}>上传</Text>
          )}
        </TouchableOpacity>
      </View>
    </View>
  );
}
```

#### API 服务封装

**mobile/src/services/api/userApi.ts**

```typescript
import api from './api';

export type User = {
  id: string;
  device_id: string | null;
  email: string | null;
  nickname: string | null;
  avatar_url: string | null;
  auth_type: string;
  created_at: string;
  updated_at: string;
};

export type UserUpdateParams = {
  nickname?: string;
  avatar_url?: string;
};

export const userApi = {
  // 获取当前用户信息
  getCurrentUser: async (): Promise<User> => {
    const response = await api.get('/users/me');
    return response.data;
  },

  // 更新当前用户信息
  updateCurrentUser: async (params: UserUpdateParams): Promise<User> => {
    const response = await api.patch('/users/me', params);
    return response.data;
  },

  // 通过用户ID查询
  getUserById: async (userId: string): Promise<User> => {
    const response = await api.get(`/users/${userId}`);
    return response.data;
  },

  // 通过用户名称查询
  getUserByUsername: async (username: string): Promise<User> => {
    const response = await api.get(`/users/by-username/${username}`);
    return response.data;
  },

  // 通过用户昵称搜索
  searchUsersByNickname: async (
    nickname: string,
    page: number = 1,
    pageSize: number = 20
  ): Promise<{ users: User[]; total: number }> => {
    const response = await api.get(
      `/users/by-nickname/${nickname}`,
      { params: { page, page_size: pageSize } }
    );
    return response.data;
  },
};
```

## ⚠️ 风险与缓解

### 风险1: 用户名称唯一性约束

**描述：** username 字段设置为 unique，可能导致注册冲突。

**缓解措施：**
- ✅ 注册时检查 username 是否已存在
- ✅ 如果存在，提示用户修改或生成随机 username
- ✅ 可选：允许用户输入自定义 username

---

### 风险2: 头像存储成本

**描述：** 所有用户上传头像到 OSS 可能产生大量存储费用。

**缓解措施：**
- ✅ 限制头像大小（1MB）
- ✅ 压缩和裁剪图片
- ✅ 考虑使用 CDN 缓存
- ✅ 定期清理未使用的头像

---

### 风险3: 用户信息隐私

**描述：** 通过用户ID查询可能暴露用户隐私。

**缓解措施：**
- ✅ 不返回敏感信息（hashed_password, verification_code）
- ✅ 添加访问控制（只允许登录用户查询）
- ✅ 考虑添加隐私设置开关

---

### 风险4: 昵称冲突

**描述：** 多个用户可能使用相同的昵称。

**缓解措施：**
- ✅ nickname 不设置为 unique（允许重复）
- ✅ 用户名称（username）作为唯一标识符
- ✅ 搜索时返回所有匹配的用户
