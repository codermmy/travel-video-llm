> **版本**: v1.0
> **创建日期**: 2026-02-10
> **优先级**: P1（高）
> **状态**: 待开发
> **序列号**: Round2-03

---

## 📋 需求概述

**目标**: 优化幻灯片播放体验，让AI生成的故事与照片内容强相关，并通过照片组机制实现密集的故事覆盖，打造"动画式"的叙事体验。

**背景**:
- 当前幻灯片停留时间较长（1-2秒/张），但章节故事简短（80-120字），看1-2张照片就阅读完毕
- 大部分照片没有对应的故事内容，播放时有"空档感"
- AI生成的故事与照片内容关联性不强，经常出现照片是"男人看电脑"，故事却是"九寨沟美景"的情况

**本次迭代核心**:
1. **照片组机制**: 每2-5张照片为一组，每章3-4组
2. **微故事**: 每张照片生成15-25字的微故事
3. **强关联**: 基于真实视觉识别结果生成故事，确保故事元素都能在照片中找到依据

---

## 🎯 核心需求

### 1. 照片组划分机制

| 层级 | 定义 | 数量 | 内容 |
|------|------|------|------|
| **事件** | 一次完整的旅行记录 | 1个 | 整体故事 (200-300字) |
| **章节** | 事件的子划分 | 每章10张照片 | 章节引言 (40-50字) + 章节总结 (30-40字) |
| **照片组** | 章节的细分单元 | 每章3-4组，每组2-5张 | 组主题 + 组情绪 + 场景描述 |
| **照片** | 单张照片 | - | 微故事 (15-25字) + 情绪标签 |

### 2. 故事与照片强关联

**生成流程**:
```
第一步：视觉识别
  → 对每张照片进行视觉分析
  → 提取：人物、物品、场景、动作、情绪、环境

第二步：照片组聚合
  → 将2-5张照片的识别结果聚合
  → 提取共同主题、场景演变、情绪变化

第三步：故事生成
  → 基于真实的视觉识别结果生成故事
  → 确保故事元素都能在照片中找到依据
  → 禁止编造照片中不存在的内容
```

**示例对比**:

| 场景 | 错误示例（当前） | 正确示例（目标） |
|------|-----------------|-----------------|
| 照片内容 | 男人+电脑+水杯+手机+雪天 | 同左 |
| 生成故事 | "九寨沟冬日美景，宁静祥和" | "雪夜，他在电脑前工作，拿起水杯，拿起手机给远方的她发去消息" |

### 3. 内容层级定义

| 层级 | 存储位置 | 长度 | 用途 | 展示时机 |
|------|---------|------|------|---------|
| **事件整体故事** | `events.full_story` | 200-300字 | 事件详情页完整描述 | 事件详情页 |
| **章节引言** | `event_chapters.chapter_intro` | 40-50字 | 章节开始引导 | 章节第一张照片，展示3秒 |
| **组主题** | `photo_groups.group_theme` | 10-15字 | 照片组主题标题 | 组内第一张照片时展示 |
| **微故事** | `photos.micro_story` | 15-25字 | 单张照片叙事 | 每张照片展示时同步显示 |
| **情绪标签** | `photos.emotion_tag` | 1个标签 | 照片情绪 | 用于播放效果和滤镜 |
| **章节总结** | `event_chapters.chapter_summary` | 30-40字 | 章节收尾 | 章节最后一张照片，展示2秒 |

## 📊 现状问题分析

### 问题1: 故事与照片内容脱节

**当前实现** (`backend/app/integrations/providers/openai_responses.py`):
```python
prompt = f"""请根据以下信息生成一个简短的旅行故事：

地点：{location}
时间：{date_range}
照片描述：
{desc_text}  # 只基于3-5张图片的简单描述

要求：
1. 100-200字
2. 情感基调：宁静、放松
...
"""
```

**问题**:
- 地点信息可能不准确（如照片是在家里，却用了GPS位置）
- 照片描述过于简单，AI容易"自由发挥"
- 没有约束AI不得编造内容

### 问题2: 内容密度不足

**当前结构**:
```
50张照片 → 3个章节（每章~17张）→ 3段故事（每段80-120字）
```

**问题**:
- 章节太少，大部分照片没有对应故事
- 看完章节故事后，还有十几张照片没有新内容
- 播放节奏不匹配：故事3秒读完，照片却要播放20秒

### 问题3: 缺少视觉识别

**当前流程**:
```
上传照片 → 直接生成故事
```

**问题**:
- 没有独立的视觉识别步骤
- 无法确保故事内容与照片一致
- 缺少结构化的视觉数据存储

#### 2.2 视觉识别 Prompt

**新增文件**: `backend/app/services/vision_analysis_service.py`

```python
def analyze_photo(self, photo_url: str) -> dict[str, Any] | None:
    """分析单张照片的视觉内容"""

    prompt = """请仔细观察这张照片，提取以下信息：

1. 人物信息：
   - 性别（男/女/无法判断）
   - 大致年龄（儿童/青年/中年/老年）
   - 数量
   - 主要动作
   - 面部表情

2. 物品信息：
   - 列出所有可见的物品（电脑、手机、水杯、书籍等）
   - 物品的状态（打开/关闭、使用中/闲置等）

3. 场景信息：
   - 室内/室外
   - 具体环境（办公室/卧室/公园/餐厅等）
   - 光线条件

4. 环境信息：
   - 天气（如果可见）
   - 时间段（白天/傍晚/夜晚）
   - 整体氛围

5. 其他值得注意的细节

⚠️ 重要：
- 只描述照片中确实存在的内容
- 不要推测或添加照片中没有的信息
- 如果某项信息无法判断，请标注"无法判断"

请以JSON格式返回：
{
  "people": {...},
  "objects": [...],
  "scene": {...},
  "environment": {...},
  "other_notes": "..."
}
"""

    # 调用 Vision API (GPT-4o 或 Claude Vision)
    response = self.vision_client.analyze_image(photo_url, prompt)
    return response
```

#### 2.3 照片组聚合 Prompt

```python
def aggregate_photo_group(
    self,
    photo_analyses: list[dict],
) -> dict[str, Any] | None:
    """聚合照片组，提取共同主题"""

    analyses_text = "\n".join([
        f"照片{i+1}: {json.dumps(a, ensure_ascii=False)}"
        for i, a in enumerate(photo_analyses)
    ])

    prompt = f"""基于以下照片的视觉分析结果，总结这组照片的共同特征：

{analyses_text}

请分析：

1. 共同场景：
   - 这些照片是在同一个场景吗？
   - 场景有什么特点？

2. 人物状态：
   - 人物在做什么？
   - 情绪有什么变化？

3. 场景演变：
   - 从第一张到最后一张，场景/动作有什么变化？
   - 是否有时间推进的感觉？

4. 整体氛围：
   - 这组照片给观众的感受是什么？

请以JSON格式返回：
{{
  "common_scene": "共同场景描述",
  "person_activity": "人物活动描述",
  "scene_evolution": "场景演变描述",
  "overall_mood": "整体氛围",
  "suggested_emotion": "建议情绪标签 (Joyful/Peaceful/Exciting/Adventurous/Epic/Romantic/Nostalgic/Thoughtful/Melancholic/Solitary)"
}}
"""

    response = self.generate_completion(prompt)
    return response
```

#### 2.4 微故事生成 Prompt

```python
def generate_micro_story(
    self,
    photo_analysis: dict,
    group_context: dict,
    photo_index_in_group: int,
) -> str | None:
    """为单张照片生成微故事"""

    analysis_text = json.dumps(photo_analysis, ensure_ascii=False)
    group_context_text = json.dumps(group_context, ensure_ascii=False)

    prompt = f"""基于这张照片的真实内容，生成一个微故事。

【这张照片的视觉识别结果】
{analysis_text}

【所在照片组的上下文】
{group_context_text}
这是该组中的第 {photo_index_in_group + 1} 张照片

【严格要求】
1. 字数：15-25个中文字
2. 内容：只描述照片中确实存在的元素和场景
3. 禁止：编造照片中没有的人物、物品、场景
4. 禁止：添加与照片无关的描述
5. 风格：简洁、优美、有画面感
6. 情绪：根据照片内容自然流露

【反例】
照片内容：男人、电脑、手机、水杯、窗外下雪
❌ 错误："九寨沟冬日美景，宁静祥和"（完全无关）
❌ 错误："他兴高采烈地在海边奔跑"（照片中没有海边，没有奔跑）

【正例】
✅ 正确："雪夜，他在电脑前工作，拿起水杯，望向窗外发呆"
✅ 正确："电脑屏幕的光映在脸上，专注而认真"

请直接返回微故事文本，不需要JSON格式。
"""

    response = self.generate_completion(prompt, max_tokens=100)
    return response.strip('"')
```

#### 2.5 章节引言/总结生成 Prompt

```python
def generate_chapter_intro_summary(
    self,
    chapter_index: int,
    total_chapters: int,
    groups: list[PhotoGroup],
) -> dict[str, str] | None:
    """生成章节引言和总结"""

    # 汇总所有照片组的信息
    groups_summary = []
    for group in groups:
        groups_summary.append({
            "theme": group.group_theme,
            "emotion": group.group_emotion,
        })

    prompt = f"""你是一位旅行故事作家，请为一个旅行的特定阶段创作引言和总结。

【章节信息】
这是整个旅行的第 {chapter_index} / {total_chapters} 部分

【该章节包含的照片组】
{json.dumps(groups_summary, ensure_ascii=False)}

【创作要求】

引言 (40-50字)：
- 简短、优美
- 引出该章节的主要内容
- 有画面感

总结 (30-40字)：
- 收束该章节的情感
- 简洁有力

请以JSON格式返回：
{{
  "intro": "章节引言 (40-50字)",
  "summary": "章节总结 (30-40字)"
}}
"""

    response = self.generate_completion(prompt, max_tokens=200)
    return response
```

```
     Photo0    Photo1    Photo2    (下一组)
     微故事     微故事     微故事
     1.5秒      1.5秒      1.5秒
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  照片组1.2开始 ... (同上)                                    │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│  章节1结束                                                   │
│  → 显示章节总结 (2秒)                                        │
└─────────────────────────────────────────────────────────────┘
                          ↓
                   (进入下一章节)
```

#### 3.2 前端展示示例

**文件**: `mobile/src/components/slideshow/SlideshowPlayer.tsx`

```typescript
// 获取当前照片对应的章节、组、微故事
const getCurrentContent = (photoIndex: number) => {
  const chapter = chapters.find(ch =>
    photoIndex >= ch.photoStartIndex && photoIndex <= ch.photoEndIndex
  );

  const group = photoGroups.find(g =>
    photoIndex >= g.photoStartIndex && photoIndex <= g.photoEndIndex
  );

  const photo = photos[photoIndex];

  return { chapter, group, photo };
};

// 显示逻辑
const { chapter, group, photo } = getCurrentContent(currentIndex);

// 文案优先级：
// 1. 章节开始 → 显示章节引言
// 2. 组开始 → 显示组主题 + 微故事
// 3. 普通照片 → 显示微故事
// 4. 章节结束 → 显示章节总结

const displayContent = useMemo(() => {
  const isFirstPhotoInChapter = currentIndex === chapter?.photoStartIndex;
  const isLastPhotoInChapter = currentIndex === chapter?.photoEndIndex;
  const isFirstPhotoInGroup = currentIndex === group?.photoStartIndex;

  if (isFirstPhotoInChapter && chapter?.chapterIntro) {
    return {
      type: 'chapter-intro',
      text: chapter.chapterIntro,
      duration: 3000,
    };
  }

  if (isLastPhotoInChapter && chapter?.chapterSummary) {
    return {
      type: 'chapter-summary',
      text: chapter.chapterSummary,
      duration: 2000,
    };
  }

  // 组主题 + 微故事
  const microStory = photo?.microStory || '';
  const groupTheme = isFirstPhotoInGroup ? group?.groupTheme : '';

  if (microStory || groupTheme) {
    return {
      type: 'micro-story',
      groupTheme,  // 仅在组第一张显示
      microStory,  // 每张都显示
      duration: 1500,
    };
  }

  return null;
}, [currentIndex, chapter, group, photo]);
```
