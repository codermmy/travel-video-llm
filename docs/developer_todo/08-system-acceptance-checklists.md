# Stage-08 系统测试与优化清单

## 1) 功能测试清单

### 认证模块
- [ ] 设备注册：首次启动可成功注册并保存 token
- [ ] 邮箱注册：可使用邮箱 + 密码 + 昵称创建账号
- [ ] 邮箱登录：可使用邮箱密码登录并进入主页面
- [ ] Token 失效：过期 token 会触发登出并跳转认证页

### 照片导入与上传
- [ ] 相册权限：拒绝时显示引导，授权后可继续流程
- [ ] EXIF 提取：含 GPS/时间信息的照片可正确解析
- [ ] 去重校验：重复 hash 不会重复上传
- [ ] 上传进度：展示 uploading/processing/completed/failed 状态
- [ ] 后台继续：点击“Continue in Background”后任务继续执行

### 事件与详情
- [ ] 事件列表：支持分页、下拉刷新、加载更多
- [ ] 详情页：封面、标题、地点、时间、故事、音乐信息展示正确
- [ ] 照片网格：3 列展示，空态提示正常
- [ ] 照片查看器：可全屏滑动查看并显示时间/GPS 元数据
- [ ] 幻灯片播放：支持自动播放、暂停/继续、上一张/下一张、退出

## 2) 性能测试清单

### 指标目标
- 冷启动：中端设备 < 2 秒
- 扫描 1000 张照片：< 60 秒
- 上传 100 张（200KB）：4G 网络 < 30 秒
- 幻灯片播放内存：长期运行不超过 200MB

### 执行方式
- Android 使用 Android Studio Profiler 采集 CPU/内存曲线
- iOS 使用 Xcode Instruments 采集内存泄漏与渲染帧率
- 网络通过系统 Network Link Conditioner 模拟 3G/4G/Wi-Fi

## 4) 优化建议清单

### 性能优化
1. 为事件详情页启用照片分页加载（当前已具备网格虚拟化，可继续后端分页）
2. 对幻灯片资源做预加载策略，优先加载前后各一张
3. 上传流程增加失败分片重试与退避策略

### 体验优化
1. 上传完成后增加本地通知（后台继续场景）
2. 音乐切换能力与音乐列表选择页联动
3. 幻灯片支持手势左右滑动并显示控制层引导
