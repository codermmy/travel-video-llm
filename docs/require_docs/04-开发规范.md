## 1. 命名规范

### 1.1 通用原则

- **KISS 原则**：命名应简洁明了，避免缩写和缩写词
- **一致性**：同一概念使用同一术语，不使用同义词
- **语义化**：命名应描述"做什么"而非"怎么做"

#### 分支类型说明

| 分支类型 | 命名格式 | 用途 | 合并目标 |
|---------|---------|------|----------|
| `main` | - | 生产环境，保持稳定 | 接受 `develop` 合并 |
| `develop` | - | 开发环境，集成最新功能 | 接受所有 feature/bugfix |

| `bugfix/*` | `bugfix/问题描述` | Bug 修复 | 合并到 `develop` |
| `hotfix/*` | `hotfix/紧急问题` | 生产环境紧急修复 | 合并到 `main` 和 `develop` |

  // 服务器错误
  const status = error.response.status;

class AppException(Exception):

    def __init__(self, message: str, code: str = "INTERNAL_ERROR"):
        self.message = message
        self.code = code
        super().__init__(message)

class PhotoNotFoundException(AppException):
    """照片不存在异常"""

    def __init__(self, photo_id: int):
        super().__init__(
            message=f"照片 {photo_id} 不存在",
            code="PHOTO_NOT_FOUND",
        )

# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
# 全局异常处理器
# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
@app.exception_handler(AppException)
async def app_exception_handler(request: Request, exc: AppException):
    return JSONResponse(
        status_code=400,
        content={
            "success": False,
            "error": {
                "code": exc.code,
                "message": exc.message,
            },
            "timestamp": datetime.utcnow().isoformat(),
        },
    )

# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

@router.get("/photos/{photo_id}")
async def get_photo(photo_id: int, db: Session = Depends(get_db)):

    if not photo:
