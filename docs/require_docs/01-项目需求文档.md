## 1. 项目概述

### 1.1 项目背景

随着智能手机的普及，每个人手机相册中都存储着海量照片。这些照片记录了生活中的美好瞬间，但往往杂乱无章，被永久封存在手机深处，难以再次翻看。特别是旅行中的照片，散落在相册的不同时间位置，用户缺乏有效的工具将它们串联成完整的故事。

**核心痛点**：
- 📸 照片数量庞大，整理困难
- 🗺️ 旅行照片散落，缺乏时空关联
- 🎬 缺乏自动化的故事化呈现工具
- 🎵 无法为旅行回忆匹配合适的配乐

### 1.2 项目目标

研发一款**移动端应用程序**，通过人工智能技术，自动发现并聚合用户相册中的零散旅行片段，将其串联成专属的旅程影片和旅途回忆。

**核心能力**：
1. 🤖 **智能聚合**：基于时间和地理位置自动聚类旅行事件
2. 📍 **轨迹可视化**：在地图上展示旅行足迹
3. 📖 **故事生成**：AI自动生成旅行故事和标题
4. 🎵 **智能配乐**：根据照片情感匹配个性化音乐
5. 🎬 **幻灯片播放**：自动生成带配乐的幻灯片视频

### 1.3 项目特色

- ✅ **全自动化**：用户只需导入相册，系统自动完成整理、分类、配乐、成片
- ✅ **增量同步**：只上传新照片，节省流量和时间
- ✅ **跨模态AI**：融合视觉理解和自然语言生成技术
- ✅ **零成本运行**：利用各类服务的免费额度，几乎无运行成本

### 1.4 成果形式

- ✅ 可运行的移动端应用（iOS/Android）
- ✅ 毕业设计论文
- ✅ 完整的源代码和实验结果
- ✅ 答辩PPT和系统演示视频

### 2.2 核心功能详细说明

#### 2.2.1 用户认证模块

**功能描述**：
提供用户身份认证功能，实现数据隔离和跨设备同步。采用简化的设备注册机制，用户无需复杂的密码管理即可使用。

**设计目标**：
- 🔒 **数据隔离**：确保每个用户只能访问自己的数据，防止误操作他人数据
- 🔄 **跨设备同步**：用户换设备后可查看之前整理的旅行记录
- 🚀 **简化流程**：无需密码/手机号注册，降低使用门槛
- 💾 **数据持久化**：云端保存事件、故事、缩略图等生成内容

**功能细节**：

1. **首次启动注册**
   - 应用首次启动时生成设备唯一标识（UUID）
   - 引导用户设置昵称（可选）
   - 调用后端注册接口，创建用户账号
   - 后端返回 JWT Token，存储到本地 AsyncStorage

2. **自动登录**
   - 后续启动时检查本地是否有有效 Token
   - 有 Token：直接使用，自动登录
   - 无 Token/Token 过期：引导用户重新注册

3. **Token 管理**
   - Access Token 有效期：30 天（毕业设计可适当延长）
   - Token 存储位置：AsyncStorage（前端）
   - 请求携带方式：HTTP Header `Authorization: Bearer <token>`
   - Token 过期处理：自动重新注册或提示用户

4. **数据隔离**
   - 所有 API 请求携带 Token，后端解析获取 user_id
   - 数据库查询强制带 user_id 过滤条件
   - 照片、事件表都关联 user_id

5. **跨设备同步**
   - 用户可在新设备上使用相同的设备 ID 登录（需手动输入或扫码）
   - 或支持多个设备 ID 绑定同一账号（可选）
   - 云端数据（缩略图、事件、故事）在所有设备间同步

**认证流程图**：

```

│                       认证流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │

│    │                                                         │
│    ├─ 生成设备 ID (UUID)                                    │
│    ├─ 显示注册页面，输入昵称（可选）                         │
│    ├─ POST /api/v1/auth/register                            │
│    │   请求体: { deviceId, nickname }                       │
│    │   返回: { userId, token, deviceId }                   │
│    ├─ 存储 Token 到本地                                     │
│    └─ 进入应用主页                                          │
│                                                              │
│  后续启动                                                    │
│    │                                                         │
│    ├─ 检查本地 Token                                        │
│    ├─ Token 有效 → 自动登录，进入主页                       │
│    └─ Token 无效 → 显示注册页面                             │
│                                                              │

**跨设备数据访问**：

```

#### 2.2.2 用户权限管理

**功能描述**：
调用Android/iOS系统API申请相册访问权限，确保系统能够读取照片。

**功能细节**：

1. **首次授权**
   - 应用首次启动时调用系统API申请相册权限
   - Android/iOS系统弹出标准的权限请求对话框（系统UI，应用无法自定义）
   - 用户点击"允许"后，系统自动记住授权决定

2. **权限检查与引导**
   - 每次启动时检查权限状态（调用系统API）
   - 如果权限被拒绝，显示引导页面提示用户手动开启
   - 引导文案："请在 设置 → 隐私 → 相册 中开启访问权限"

**说明**：
- Android/iOS系统会自动记住用户的授权决定，无需应用层实现持久化
- 采用系统标准权限流程，符合主流应用的交互习惯

**验收标准**：
- [ ] 首次启动能调用系统API弹出权限请求
- [ ] 用户授权后，系统能正常访问相册
- [ ] 权限被撤销后，能正确显示引导提示

#### 2.2.3 照片管理模块

**功能描述**：
扫描用户本地相册，提取照片元数据，生成缩略图，并增量上传到云端。

**功能细节**：

1. **相册扫描**
   - 扫描手机相册中的所有照片
   - 支持格式：JPEG, PNG, HEIC（iOS）
   - 过滤视频文件（暂不处理）

2. **元数据提取**
   - 提取EXIF信息：
     - 📍 GPS坐标（经度、纬度）
     - 🕐 拍摄时间（DateTimeOriginal）
     - 📷 相机型号（可选）
   - 计算照片SHA-256哈希值（用于去重）

3. **缩略图生成**
   - 原缩比例：宽度统一为1080px，高度自适应
   - 压缩质量：JPEG 80%
   - 目标大小：约200KB/张
   - 保存到本地缓存目录

4. **增量检测**
   - 对比本地照片哈希与云端已上传列表
   - 识别新增照片
   - 识别已删除照片（用户从本地删除，但云端有）

5. **云端上传**
   - 只上传新照片的缩略图
   - 上传元数据：哈希、GPS、拍摄时间、本地路径
   - 支持断点续传（如果上传中断）
   - 显示上传进度

**数据结构**：

```json
{
  "hash": "a3d5f8e9b2c1...",        // SHA-256哈希
  "localPath": "/storage/...",      // 本地路径
  "thumbnailPath": "/cache/...",    // 缩略图本地路径
  "originalUrl": "https://oss...",  // 原图URL（可选，暂时不存原图）
  "thumbnailUrl": "https://oss...", // 缩略图云端URL
  "gpsLat": 30.2591727,            // 纬度
  "gpsLon": 120.2152914,           // 经度
  "shootTime": "2024-10-01T10:30:00Z",  // 拍摄时间
  "uploadTime": "2024-10-15T08:00:00Z", // 上传时间
  "eventId": null                   // 所属事件ID（聚类后填充）
}
```

**验收标准**：
- [ ] 能正确扫描相册中的所有照片
- [ ] 能准确提取EXIF信息
- [ ] 缩略图大小控制在150-250KB之间
- [ ] 增量检测准确，不会重复上传
- [ ] 上传进度实时显示

#### 2.2.4 智能聚类模块

**功能描述**：
基于时间和地理位置，自动将照片聚类为"旅行事件"，如"杭州之旅"、"北京之行"。

**功能需求**：

1. **输入输出**
   - **输入**：一组照片（包含GPS坐标、拍摄时间）
   - **输出**：若干个事件（每个事件包含多张相关联的照片）

2. **聚类规则**
   - 时间相近的照片应归为同一事件
   - 地理位置相近的照片应归为同一事件
   - 一个事件至少包含一定数量的照片（具体数量待定）

3. **事件生成**
   - 为每个聚类生成一个事件
   - 事件属性：
     - 开始时间：第一张照片的拍摄时间
     - 结束时间：最后一张照片的拍摄时间
     - 持续天数：自动计算
     - 中心坐标：所有照片GPS的平均值

4. **逆向地理编码**
   - 调用高德地图API
   - 将GPS坐标转换为地址：`30.259, 120.215` → `"浙江省杭州市西湖区"`
   - 提取城市名和区县信息

5. **封面挑选**
   - 从事件的照片中自动选择一张作为封面
   - 选择标准：图像清晰度高、构图质量好

6. **增量合并**
   - 新上传的照片能自动归入现有事件
   - 更新事件的结束时间和照片数量

7. **边界情况处理**
   - 无GPS信息的照片如何处理
   - 跨时区的旅行如何处理
   - 短时间内拍摄的大量照片如何处理

**📌 注**：
具体的聚类算法设计（算法选型、参数调优、边界情况处理）将在《03-聚类算法设计文档》中详细说明。
当前算法方案待定，将在开发阶段通过实验确定最佳方案。

**验收标准**：
- [ ] 算法能准确识别同一旅行的照片
- [ ] 生成的事件符合用户直觉（如"杭州3日游"算一个事件）
- [ ] 同一旅行的照片不会被错误拆分成多个事件
- [ ] 不同旅行的照片不会被错误合并
- [ ] 逆向地理编码准确率 > 95%
- [ ] 封面照片质量良好

#### 2.2.5 AI内容生成模块

**功能描述**：
利用AI技术，自动为每个旅行事件生成标题、故事文本和情感标签。

**功能细节**：

1. **图像内容识别**
   - 模型：通义万相（阿里云）
   - 输入：事件的封面照片
   - 输出：
     - 场景描述："西湖边的断桥，周围有绿色的柳树，天空晴朗"
     - 关键词：["湖泊", "桥", "风景", "自然", "户外"]
     - 情感基调：初步判断（"calm", "happy", "epic"）

2. **批量照片分析**
   - 对于事件的第1、5、10张照片（抽样）
   - 生成简短描述
   - 汇总为：`["西湖边的雷峰塔", "灵隐寺的佛像", "断桥上的游客"]`

3. **故事文本生成**
   - 模型：通义千问（阿里云）
   - 输入：
     ```
     请根据以下信息生成一个简短的旅行故事：

     地点：杭州西湖
     时间：2024年10月1日-3日
     照片描述：
     - 西湖边的断桥，周围有柳树
     - 雷峰塔日落景色
     - 灵隐寺的古建筑
     - 湖边散步的人们

     要求：
     1. 100-200字
     2. 情感基调：宁静、放松
     3. 包含地点信息
     4. 语言优美，有故事感
     ```
   - 输出：
     ```
     🍂 秋日杭州之旅

     10月的西湖，秋意渐浓。你漫步在断桥之上，两岸的柳树在微风中轻摆。

     雷峰塔下，夕阳将湖面染成金色。灵隐寺的钟声悠远，让这趟旅程多了几分宁静。

     三天的时光，在湖光山色中缓缓流淌。
     ```

4. **标题生成**
   - 模型：通义千问
   - 输入：地点 + 时间 + 情感
   - 输出：`"杭州西湖三日游"` 或 `"秋日杭州之旅"`

5. **情感标签推理**
   - 模型：通义千问
   - 输入：所有照片的描述
   - 输出：四个情感标签之一
     - `"Happy"` - 欢快（多人、笑容、鲜艳色彩）
     - `"Calm"` - 宁静（自然风光、独处、冷色调）
     - `"Epic"` - 史诗感（壮丽风景、建筑、广角）
     - `"Romantic"` - 浪漫（日落、花朵、情侣）

**API调用示例**：

```javascript
// 图像理解

// 返回：

  "description": "西湖边的断桥，周围有绿色的柳树",

  "title": "秋日杭州之旅",
  "story": "10月的西湖，秋意渐浓...",

}
```

**验收标准**：
- [ ] 图像描述准确率 > 85%
- [ ] 故事文本通顺、有故事感
- [ ] 标题简洁明了
- [ ] 情感标签与照片内容匹配

#### 2.2.6 音乐匹配模块

**功能描述**：
根据事件的情感标签，从音乐库中智能匹配合适的背景音乐。

**功能细节**：

1. **音乐库管理**
   - 来源：Pixabay Music（免版权）
   - 数量：200首（每个情感分类50首）
   - 存储：上传到阿里云OSS
   - 格式：MP3，128kbps

2. **情感分类**
   - **Happy**（欢快）：50首
     - 特征：快节奏（120-140 BPM）、明亮调式、流行/轻音乐
   - **Calm**（宁静）：50首
     - 特征：慢节奏（60-80 BPM）、自然声音、钢琴/纯音乐
   - **Epic**（史诗感）：50首
     - 特征：管弦乐、宏大、渐强
   - **Romantic**（浪漫）：50首
     - 特征：钢琴、吉他、温柔旋律

3. **匹配算法**
   - 输入：事件情感标签（如 `"Calm"`）
   - 处理：
     1. 从对应分类中随机选择一首
     2. 或根据照片数量选择时长合适的音乐
     3. 记录选择结果，避免重复（同一事件多次刷新时保持同一首）
   - 输出：音乐URL

4. **音乐更换**
   - 用户可以"换一首"功能
   - 从同一情感分类中随机选择另一首
   - 最多循环展示10首

**数据结构**：

```json
{
  "musicId": "calm_piano_01",
  "title": "Morning Calm",
  "emotion": "Calm",
  "duration": 180,           // 时长（秒）
  "bpm": 70,                // 节奏
  "url": "https://oss.../calm_piano_01.mp3",
  "artist": "Pixabay Artist"
}
```

**验收标准**：
- [ ] 音乐情感与照片内容匹配
- [ ] 音乐播放流畅
- [ ] "换一首"功能正常工作
- [ ] 音乐均为免版权，可商用

#### 3.1.0 开发环境 vs 生产环境

**开发策略**：采用"本地开发优先"策略，前期在本地完成所有功能开发和测试，功能完成后再考虑部署到云端。

**开发环境（前期）**：
- 后端：本地运行（localhost:8000）

- 文件存储：本地文件系统（./uploads/）
- 对象存储：暂不使用，直接存本地

**生产环境（后期部署）**：
- 后端：云服务器（腾讯云轻量服务器）
- 数据库：云数据库（腾讯云PostgreSQL）
- 文件存储：阿里云OSS（推荐，5GB免费）或服务器本地存储
- AI服务：通义千问API（直接调用，无需部署）
- 地图服务：高德地图API（直接调用，无需部署）

**优先级**：
- 🔴 高优先级：本地开发环境搭建
- 🟢 低优先级：云端部署（功能完成后再考虑）

### 3.3 数据模型设计

#### 3.3.1 数据库表结构

```sql
-- 用户表
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    phone VARCHAR(20) UNIQUE,
    device_id VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 照片表
CREATE TABLE photos (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    file_hash VARCHAR(64) UNIQUE NOT NULL,     -- SHA-256哈希
    local_path VARCHAR(500),                    -- 本地路径
    thumbnail_url VARCHAR(500),                 -- 缩略图URL
    gps_lat DECIMAL(10, 7),                     -- 纬度
    gps_lon DECIMAL(10, 7),                     -- 经度
    shoot_time TIMESTAMP,                       -- 拍摄时间
    upload_time TIMESTAMP DEFAULT NOW(),        -- 上传时间
    event_id INTEGER REFERENCES events(id),     -- 所属事件
    status VARCHAR(20) DEFAULT 'uploaded',      -- uploaded/clustered
    created_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_photos_hash ON photos(file_hash);
CREATE INDEX idx_photos_event ON photos(event_id);

-- 事件表
CREATE TABLE events (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    title VARCHAR(200),                          -- AI生成的标题
    location_name VARCHAR(100),                  -- 逆向地理编码结果
    gps_lat DECIMAL(10, 7),                      -- 中心坐标
    gps_lon DECIMAL(10, 7),
    start_time TIMESTAMP,                        -- 开始时间
    end_time TIMESTAMP,                          -- 结束时间
    photo_count INTEGER DEFAULT 0,               -- 照片数量
    cover_photo_url VARCHAR(500),                -- 封面URL
    story_text TEXT,                             -- AI生成的故事
    emotion_tag VARCHAR(20),                     -- Happy/Calm/Epic/Romantic
    music_url VARCHAR(500),                      -- 音乐URL
    music_id VARCHAR(100),                       -- 音乐ID
    status VARCHAR(20) DEFAULT 'clustered',      -- clustered/generated
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_events_user ON events(user_id);
CREATE INDEX idx_events_time ON events(start_time DESC);
```

具体的API接口设计将在后续的技术方案评审中确定，届时将补充详细的接口文档。

### A.1 云服务选型

| 服务 | 选型 | 理由 |

| **数据库托管** | 腾讯云PostgreSQL | 按量付费，灵活 |

### A.2 本地到云端迁移方案

#### A.2.1 需要迁移的组件

| 组件 | 本地开发 | 云端部署 | 迁移难度 |
|------|---------|---------|----------|
| 后端API | localhost:8000 | 云服务器 | 🟢 简单 |
| 数据库 | 本地PostgreSQL | 云数据库或本地 | 🟢 简单 |
| 文件存储 | ./uploads/ | 阿里云OSS | 🟡 中等 |
| Redis | 本地Redis | 服务器本地Redis | 🟢 简单 |

#### A.2.2 代码改动点

**存储服务抽象化**：

```python
# ❌ 开发阶段：直接写本地路径
file_path = f"./uploads/{photo_id}.jpg"

# ✅ 上线时：改成抽象接口
class StorageService:
    async def save(self, file: UploadFile, path: str) -> str:
        # 开发环境：存本地
        # 生产环境：存OSS
        pass
```

#### A.2.3 部署步骤

1. **购买云服务器**（腾讯云/阿里云）
2. **安装运行环境**（Python, PostgreSQL, Redis）
3. **上传代码**（git pull 或 rsync）
4. **配置环境变量**（.env文件）
5. **启动服务**（使用systemd或supervisor）
6. **配置域名（可选）**

### A.3 成本估算

#### 月度成本

| 项目 | 月成本 | 说明 |
|------|--------|------|
