# 时空聚类功能详解

## 1. 功能概述

旅行事件聚合模块是本系统的核心模块之一，其目标是将零散的照片组织成更贴近真实旅途记忆的事件片段。系统采用**纯规则的时空聚类方案**，不依赖任何机器学习聚类算法（如 DBSCAN、HDBSCAN），而是根据旅行场景的特点设计直观的规则逻辑。

### 1.1 核心特点

| 特点 | 说明 |
|------|------|
| 无 ML 算法 | 不使用 DBSCAN/HDBSCAN，不构建距离矩阵，不使用向量嵌入 |
| 线性扫描 | 按时间排序后单次扫描，算法复杂度 O(n) |
| 规则可解释 | 用户能理解"为什么这些照片被分成一个事件" |
| 参数直观 | 阈值含义明确，如 48 小时时间间隙、200km 跨城距离等 |
| 后处理丰富 | 6 层规则后处理，适应各种旅行场景 |

### 1.2 与传统聚类方法的区别

传统照片聚类方法通常采用机器学习算法：

| 方法 | 原理 | 局限性 |
|------|------|--------|
| K-Means | 需预先指定簇数量 | 旅行事件数量未知 |
| DBSCAN | 基于密度聚类 | 需构建 O(n²) 距离矩阵，参数难调 |
| HDBSCAN | 自适应 DBSCAN | 同样需要距离矩阵，对旅行场景过复杂 |

本系统的规则方案更贴合旅行场景特点：

1. **时间线性特征**：旅行照片天然按拍摄时间有序，不需要发现复杂形状的簇
2. **事件边界明确**：用户直觉上用"时间间隙"和"地点跳跃"判断事件边界
3. **效率要求**：移动端导入后需要快速响应，O(n) 比 O(n²) 更合适

## 3. 核心聚类算法详解

### 3.1 RuleBasedClustering 类设计

**位置**：`backend/app/services/clustering_service.py:128-182`

```python
class RuleBasedClustering:
    """纯规则聚类：扫描分割点，无 ML 算法。

    设计原理：
    - 旅行照片天然按时间有序，不需要复杂聚类
    - 分割判断基于直观条件：时间间隙、空间跳跃
    - 算法复杂度 O(n)，比 DBSCAN 的 O(n²) 更高效
    """

    def __init__(self, config: ClusteringConfig) -> None:
        self.config = config

    def cluster(self, photos: list[PhotoData]) -> list[list[PhotoData]]:
        """执行规则聚类，返回照片簇列表。"""
        if not photos:
            return []

        # Step 1: 按时间排序（核心假设：旅行照片时间线性）
        ordered = sorted(photos, key=lambda p: p.shoot_time)

        # Step 2: 扫描分割点
        clusters: list[list[PhotoData]] = []
        current: list[PhotoData] = [ordered[0]]

        time_threshold = timedelta(hours=self.config.time_threshold_hours)
        distance_threshold = self.config.city_jump_threshold_km

        for photo in ordered[1:]:
            prev = current[-1]

            # 计算时间间隙
            time_gap = photo.shoot_time - prev.shoot_time

            # 计算GPS距离（仅当两张照片都有有效GPS时）
            spatial_jump = False
            if _photo_has_valid_gps(prev) and _photo_has_valid_gps(photo):
                distance = haversine_distance(
                    prev.gps_lat, prev.gps_lon,
                    photo.gps_lat, photo.gps_lon
                )

                # 分割条件 B: >200km 大跳跃（跨城/跨国）
                # 设计意图：上海到北京约1200km，无论时间间隔都应分割
                if distance > 200:
                    spatial_jump = True

                # 分割条件 C: 城市跳跃阈值 + 时间条件
                # 设计意图：50km内同城不分割，超过需同时满足时间间隔
                elif distance > distance_threshold and time_gap >= timedelta(minutes=60):
                    spatial_jump = True

            # 分割决策
            # 条件 A: 时间间隙 > threshold（如48小时）
            # 条件 B/C: 空间跳跃
            if time_gap > time_threshold or spatial_jump:
                clusters.append(current)
                current = [photo]  # 开始新簇
            else:
                current.append(photo)  # 继续当前簇

        # 添加最后一个簇
        clusters.append(current)
        return clusters
```

### 3.2 分割条件详解

| 条件 | 阈值 | 设计意图 | 适用场景 |
|------|------|---------|---------|
| **条件 A** | time_gap > 48h | 长时间间隔必然分割 | 不同日期的活动、隔天旅行 |
| **条件 B** | distance > 200km | 跨城/跨国直接分割 | 上海→北京、中国→日本 |
| **条件 C** | distance > 50km AND time_gap >= 60min | 城市跳跃需时间验证 | 成都→重庆（约300km） |

**为什么条件 C 需要时间验证？**

- 避免同一景点内的GPS漂移导致误分割
- 例如：用户在西湖边拍照，GPS可能有几十米误差，不应分割
- 又如：上午杭州西湖，下午杭州灵隐寺，距离约10km，同城不分割

**为什么条件 B 不需要时间验证？**

- 200km以上的跳跃不可能发生在短时间内
- 上海到北京约1200km，不可能10分钟内完成
- 这是"硬分割"条件，保证跨城/跨国100%分离

### 3.3 GPS 有效性判断

```python
def _has_valid_gps(lat: Optional[float], lon: Optional[float]) -> bool:
    """判断GPS坐标是否有效。

    无效情况：
    1. lat 或 lon 为 None（照片无GPS信息）
    2. lat=0.0 且 lon=0.0（GPS未初始化的默认值）
    3. 超出有效范围（lat不在[-90,90]，lon不在[-180,180]）
    """
    if lat is None or lon is None:
        return False

    # 过滤 (0, 0) 这个无效默认值
    if math.isclose(lat, 0.0, abs_tol=1e-9) and math.isclose(lon, 0.0, abs_tol=1e-9):
        return False

    return -90.0 <= lat <= 90.0 and -180.0 <= lon <= 180.0
```

**为什么过滤 (0, 0)？**

- 很多相机/手机在GPS未初始化时默认写入 (0, 0)
- 这个位置在非洲几内亚湾，不可能是中国旅行照片的地点
- 不过滤会导致错误的距离计算

## 4. 后处理规则详解

### 4.1 merge_short_intervals()

**功能**：合并短时间间隔的相邻事件

**位置**：`clustering_service.py:189-211`

```python
@staticmethod
def merge_short_intervals(
    events: list[list[PhotoData]],
    min_interval: timedelta,  # 默认 180 分钟（3小时）
) -> list[list[PhotoData]]:
    """合并时间间隔 < min_interval 的相邻事件。

    场景：用户在西湖游玩，中午吃饭休息1小时，不应被拆成两个事件。
    """
    if len(events) < 2:
        return events

    ordered = sorted(events, key=lambda photos: photos[0].shoot_time)
    merged: list[list[PhotoData]] = []

    for current in ordered:
        if not merged:
            merged.append(list(current))
            continue

        previous = merged[-1]
        gap = current[0].shoot_time - previous[-1].shoot_time

        if gap < min_interval:
            # 合并：将当前事件的照片加入前一事件
            merged[-1] = sorted(previous + current, key=lambda photo: photo.shoot_time)
        else:
            merged.append(list(current))

    return merged
```

**典型场景**：

| 场景 | 照片分布 | 处理结果 |
|------|---------|---------|
| 西湖游玩 | 10:00-11:30拍照，12:00-13:00吃饭，13:30-15:00拍照 | 合并为一个事件 |
| 博物馆参观 | 上午拍照，下午外出午餐后返回 | 合并为一个事件 |

### 4.2 split_large_internal_gaps()

**功能**：在事件内部检测大时间间隙并切分

**位置**：`clustering_service.py:213-293`

```python
@staticmethod
def split_large_internal_gaps(
    events: list[list[PhotoData]],
    max_time_gap: timedelta,           # 默认 48h
    jump_threshold_km: float,          # 默认 50km
    min_time_gap_for_jump: timedelta,  # 默认 180min
) -> list[list[PhotoData]]:
    """在事件内部检测大时间间隙并切分。

    自适应阈值设计：
    - 当事件内照片数 ≥ 80 或跨度 ≥ 10 天时启用
    - 取所有间隙的 90 分位数作为阈值
    - 限制在 12-36 小时范围内
    """
    segmented: list[list[PhotoData]] = []

    for event in events:
        if len(event) <= 1:
            segmented.append(event)
            continue

        ordered = sorted(event, key=lambda photo: photo.shoot_time)
        event_span = ordered[-1].shoot_time - ordered[0].shoot_time

        # 自适应阈值计算
        adaptive_gap: Optional[timedelta] = None
        if len(ordered) >= 80 or event_span >= timedelta(days=10):
            # 计算所有相邻照片的间隙
            gaps = [
                (ordered[idx].shoot_time - ordered[idx-1].shoot_time).total_seconds()
                for idx in range(1, len(ordered))
            ]
            # 取 90 分位数
            adaptive_seconds = np.percentile(gaps, 90)
            # 限制范围：12-36小时
            adaptive_seconds = max(12*3600, min(adaptive_seconds, 36*3600))
            adaptive_gap = timedelta(seconds=adaptive_seconds)

        # 扫描分割
        current: list[PhotoData] = [ordered[0]]
        for photo in ordered[1:]:
            previous = current[-1]
            time_gap = photo.shoot_time - previous.shoot_time

            should_split = False

            # 条件 1: 超过最大间隙
            if time_gap > max_time_gap:
                should_split = True

            # 条件 2: 超过自适应阈值
            if adaptive_gap and time_gap >= adaptive_gap:
                should_split = True

            # 条件 3: GPS跳跃 + 时间条件
            if not should_split and _photo_has_valid_gps(previous) and _photo_has_valid_gps(photo):
                distance = haversine_distance(...)
                if distance > jump_threshold_km and time_gap >= min_time_gap_for_jump:
                    should_split = True

            if should_split:
                segmented.append(current)
                current = [photo]
            else:
                current.append(photo)

        segmented.append(current)

    return segmented
```

**为什么需要自适应阈值？**

- 固定阈值对大事件不够灵活
- 例如：一个月旅行，每天拍照，间隙约24小时
- 固定48h阈值可能无法有效分割
- 90分位数能找到"异常大"的间隙

### 4.3 split_oversized_events()

**功能**：切分超过最大跨度的事件

**位置**：`clustering_service.py:332-387`

```python
@staticmethod
def split_oversized_events(
    events: list[list[PhotoData]],
    default_max_span: timedelta = timedelta(days=14),      # 默认最大跨度
    aggressive_max_span: timedelta = timedelta(days=7),    # 大事件更严格
    aggressive_span_threshold: timedelta = timedelta(days=45),
    aggressive_photo_threshold: int = 120,
) -> list[list[PhotoData]]:
    """切分超过最大跨度的事件。

    设计意图：
    - 防止"一个月旅行"被合并成一个超大事件
    - 大事件使用更严格的阈值
    - 递归分割直到每个事件都在阈值内
    """
    normalized: list[list[PhotoData]] = []

    for event in events:
        ordered_event = sorted(event, key=lambda photo: photo.shoot_time)

        # 确定阈值
        event_span = ordered_event[-1].shoot_time - ordered_event[0].shoot_time
        max_span = default_max_span
        if len(ordered_event) >= aggressive_photo_threshold or event_span >= aggressive_span_threshold:
            max_span = aggressive_max_span  # 大事件用7天阈值

        # 递归分割
        pending = [ordered_event]
        while pending:
            current = pending.pop(0)
            current_span = current[-1].shoot_time - current[0].shoot_time

            if current_span <= max_span:
                normalized.append(current)
                continue

            # 找最佳分割点
            split_index = _find_split_index_for_max_span(current, max_span)
            if split_index:
                pending.insert(0, current[split_index:])
                pending.insert(0, current[:split_index])
            else:
                normalized.append(current)

    return normalized
```

**分割点选择策略**：

- 不是简单地在阈值时间点分割
- 寻找"最自然"的分割点：间隙大且接近阈值时间
- 评分公式：`score = gap * closeness`
- 避免在连拍照片中间分割

### 4.4 split_city_transitions()

**功能**：检测城市跳跃并拆分

**位置**：`clustering_service.py:472-523`

```python
@staticmethod
def split_city_transitions(
    events: list[list[PhotoData]],
    jump_threshold_km: float,           # 默认 50km
    no_gps_merge_max_gap: timedelta = timedelta(minutes=30),
    merge_max_gap: timedelta = timedelta(hours=48),
) -> list[list[PhotoData]]:
    """检测城市跳跃并拆分。

    特殊处理：
    1. >200km 跳跃直接分割（跨城/跨国）
    2. 无GPS情况：30min内可合并，48h以上必分割
    3. 普通跳跃：超过阈值则分割
    """
    ordered = sorted(events, key=lambda photos: photos[0].shoot_time)
    result: list[list[PhotoData]] = [list(ordered[0])]

    for current in ordered[1:]:
        previous = result[-1]
        left = previous[-1]   # 前一事件最后一张
        right = current[0]    # 当前事件第一张

        time_gap = right.shoot_time - left.shoot_time
        can_measure = _photo_has_valid_gps(left) and _photo_has_valid_gps(right)

        # 条件 1: 时间间隙过大
        if time_gap > merge_max_gap:
            result.append(list(current))
            continue

        # 条件 2: 无GPS情况
        if not can_measure:
            if time_gap <= no_gps_merge_max_gap:
                # 短时间无GPS，可能是同一地点
                previous.extend(current)
                previous.sort(key=lambda photo: photo.shoot_time)
            else:
                result.append(list(current))
            continue

        # 条件 3: GPS距离计算
        distance = haversine_distance(left.gps_lat, left.gps_lon, right.gps_lat, right.gps_lon)

        # >200km 直接分割（硬规则）
        if distance > 200:
            result.append(list(current))
            continue

        # 普通跳跃阈值
        if distance > jump_threshold_km:
            result.append(list(current))
        else:
            previous.extend(current)

    return result
```

**为什么 200km 是硬分割？**

- 中国城市间距离通常 >200km
- 上海-杭州约170km，可能同城游
- 上海-南京约300km，跨城游
- 北京-天津约120km，可能同城游
- 200km 是合理的"跨城"阈值

### 4.5 merge_tiny_events()

**功能**：将照片数过少的事件合并到相邻事件

**位置**：`clustering_service.py:418-470`

```python
@staticmethod
def merge_tiny_events(
    events: list[list[PhotoData]],
    min_photos_per_event: int,          # 默认 2
    max_merge_gap: timedelta,           # 最大合并间隙
    jump_threshold_km: float,           # 跳跃阈值
) -> list[list[PhotoData]]:
    """将照片数 < min_photos_per_event 的事件合并到相邻事件。

    双向合并策略：
    1. 先向后合并（从前往后扫描）
    2. 再向前合并（从后往前扫描）
    3. 确保所有碎片事件被处理
    """
    ordered = [sorted(event, key=lambda photo: photo.shoot_time) for event in events]
    ordered.sort(key=lambda photos: photos[0].shoot_time)

    # 向后合并
    merged: list[list[PhotoData]] = []
    for current in ordered:
        if not merged:
            merged.append(list(current))
            continue

        previous = merged[-1]
        if len(current) < min_photos_per_event and _can_merge_adjacent_events(previous, current):
            previous.extend(current)
            previous.sort(key=lambda photo: photo.shoot_time)
        else:
            merged.append(list(current))

    # 向前合并（处理末尾的碎片）
    stabilized: list[list[PhotoData]] = []
    for current in reversed(merged):
        if not stabilized:
            stabilized.append(current)
            continue

        next_event = stabilized[-1]
        if len(current) < min_photos_per_event and _can_merge_adjacent_events(current, next_event):
            combined = sorted(current + next_event, key=lambda photo: photo.shoot_time)
            stabilized[-1] = combined
        else:
            stabilized.append(current)

    return list(reversed(stabilized))
```

**为什么双向合并？**

- 单向合并可能遗漏末尾碎片
- 例如：事件序列 [大, 小, 小, 大]
- 向后合并：[大+小, 小, 大] → 小仍存在
- 向前合并补充：[大+小+小, 大]

### 4.6 filter_night_singletons()

**功能**：过滤夜间单张照片

**位置**：`clustering_service.py:526-553`

```python
@staticmethod
def filter_night_singletons(events: list[list[PhotoData]]) -> list[list[PhotoData]]:
    """过滤夜间单张照片（22:00-06:00）。

    原因：
    - 夜间单张通常是误触（如睡前检查手机）
    - 无意义的噪声，不应形成事件
    - 多张照片的事件保留（可能是夜游）
    """
    if not events:
        return events

    normalized: list[list[PhotoData]] = []
    for event in events:
        # 多张照片的事件直接保留
        if len(event) != 1:
            normalized.append(event)
            continue

        # 单张照片检查时间
        hour = event[0].shoot_time.hour

        # 夜间（22:00-06:00）单张过滤
        if 22 <= hour or hour <= 6:
            continue  # 过滤掉，不加入结果

        # 白天单张保留
        normalized.append(event)

    return normalized
```

**为什么夜间定义为 22:00-06:00？**

- 符合一般作息规律
- 夜游活动通常有多张照片（如夜市、酒吧）
- 单张夜间照片大多是误触
