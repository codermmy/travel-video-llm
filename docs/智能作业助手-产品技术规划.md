# 智能作业助手 - 产品技术规划文档

> **文档版本：** v1.0
> **更新日期：** 2026-02-11
> **产品定位：** 帮助学生自动收集课程群作业、AI 辅助完成作业的工具

---

## 一、产品概述

### 1.1 目标用户

- 大学生、高中生
- 主要使用 QQ/微信 课程群进行学习交流
- 经常错过老师发布的作业信息
- 希望高效管理和完成作业

### 1.2 核心价值

1. **自动收集作业** - 从 QQ/微信 课程群中自动识别和整理作业
2. **智能提醒** - 截止日期倒计时，定时汇总推送
3. **AI 辅助** - 结合教材 PDF，AI 帮助理解和完成作业
4. **隐私保护** - 本地处理，无需老师知道，无需添加机器人

### 1.3 用户痛点

| 痛点 | 解决方案 |
|------|----------|
| 课程群消息太多，容易漏掉作业 | 自动识别作业内容，汇总展示 |
| 忘记作业截止时间 | 倒计时提醒 + 定时汇总 |
| 不理解作业要求 / 不会做 | AI 结合教材辅助解答 |
| 老师/同学不知道我在用工具 | 本地解析，不上报群聊 |

## 二、功能需求

### 2.1 核心功能（MVP）

#### 功能 1：课程群管理
- 添加/删除课程群（QQ/微信）
- 标记群内的老师/管理员
- 自动识别群主和管理员

#### 功能 2：作业收集
- 定时扫描群消息（默认每天 7:00 和 17:00）
- 手动触发同步
- LLM 识别作业内容：
  - 作业描述
  - 截止日期
  - 提交方式

#### 功能 3：作业展示
- 按截止时间排序
- 按课程分组
- 即将截止高亮
- 倒计时显示

#### 功能 4：通知推送
- 桌面端系统通知
- 新作业检测时推送
- 截止前提醒

### 2.2 扩展功能（Phase 2）

#### 功能 5：RAG 教材辅助
- 上传课程 PDF 教材
- 向量化存储
- AI 基于教材内容回答问题
- 引用教材来源（页码）

#### 功能 6：数据同步
- 云端数据存储
- Web 端查看作业
- 小程序查看 + 推送

### 2.3 可选功能（Phase 3）

#### 功能 7：自动提交作业
- MCP + 浏览器自动化
- 针对具体平台定制

## 三、技术方案

### 3.1 平台接入方案

| 平台 | 接入方式 | 可行性 | 风险 |
|------|---------|--------|------|
| **QQ** | 本地数据库解密 | ✅ 可行 | 低 |
| **微信** | 本地数据库解密 | ⚠️ 有限 | 无漫游问题 |
| **钉钉** | 机器人 API | ✅ 可行 | 需管理员权限（不推荐）|
| **飞书** | 机器人 API | ✅ 可行 | 需管理员权限（不推荐）|

**结论：** 优先支持 QQ 和微信，钉钉/飞书暂不考虑（学生群体少，且需要管理员权限）

### 3.2 微信数据采集

#### 工具选择：chatlog

```
项目地址：https://github.com/sjzar/chatlog

特点：
- 完全本地运行，不上传数据
- 支持 Windows 和 macOS
- 支持 PC 微信 3.x 和 4.0 版本
- 提供 HTTP API 和 MCP 服务
- 可解密图片、语音等多媒体
```

#### API 示例

```bash
# 获取群聊列表
GET http://localhost:5030/api/chatrooms

# 获取指定群的消息
GET http://localhost:5030/api/messages?chatroom_id=xxx&start_time=xxx
```

#### 微信消息同步问题

```
痛点：PC 微信不登录期间的消息不会同步

解决方案：
1. 引导用户定期从手机同步（聊天记录迁移）
2. 检测长时间未登录，推送提醒
3. 接受限制，优先支持 QQ（有漫游）
```

### 3.3 QQ 数据采集

#### 工具选择：qq-win-db-key 系列

```
数据库路径：
%USERPROFILE%\Documents\Tencent Files\<QQ号>\nt_qq\nt_db\nt_msg.db

解密流程：
1. 定位 QQ 数据库文件
2. 获取解密密钥（使用 qq-win-db-key 工具）
3. 解密数据库（SQLCipher）
4. 导出聊天记录
```

#### QQ 优势

- 有消息漫游（7天/30天/永久）
- 不需要保持登录也能获取历史消息
- 学生群体使用更多

### 4.3 产品形态

#### 方案：桌面端一体化 + Web 端同步

| 端 | 功能 | 说明 |
|---|------|------|
| **桌面端** | 完整功能 | 数据采集 + 作业列表 + 课程配置 + RAG 辅助 |
| **Web/小程序** | 精简功能 | 查看作业 + 接收推送 + 简单 AI 问答 |

**为什么这样设计？**
- 用户心智统一（主要在桌面端操作）
- 桌面端本来就要开着（保持微信/QQ 登录）
- 手机端只是"看一眼"的补充

## 八、核心代码逻辑

### 8.1 作业识别 Prompt

```python
HOMEWORK_IDENTIFICATION_PROMPT = """
你是一个作业识别助手。请分析以下群聊消息，识别是否包含作业信息。

群聊名称：{group_name}
发送者：{sender_name}
发送者身份：{sender_role}
消息内容：{message}
发送时间：{send_time}

请识别：
1. 是否是作业（是/否）
2. 作业内容描述
3. 截止日期（如果有）
4. 提交方式（如果有）

请以 JSON 格式返回：
{
    "is_homework": true/false,
    "content": "作业内容",
    "deadline": "YYYY-MM-DD HH:mm 或 null",
    "submit_method": "提交方式 或 null",
    "confidence": 0.0-1.0
}
```

```python
def auto_identify_teachers(group_info, messages):
    """自动识别老师/管理员"""
    candidates = []

    # 1. 群主和管理员
    candidates.append({
        "id": group_info.owner_id,
        "name": group_info.owner_name,
        "reason": "群主",
        "confidence": 0.9
    })

    for admin in group_info.admins:
        candidates.append({
            "id": admin.id,
            "name": admin.name,
            "reason": "管理员",
            "confidence": 0.7
        })

    # 2. 分析消息特征（发布作业类消息的人）
    homework_keywords = ["作业", "提交", "deadline", "DDL", "完成", "练习"]

    for sender_id, sender_messages in group_by_sender(messages).items():
        homework_count = sum(
            1 for m in sender_messages
            if any(kw in m.content for kw in homework_keywords)
        )
        if homework_count >= 3:
            candidates.append({
                "id": sender_id,
                "name": get_name_by_id(sender_id),
                "reason": f"发布过 {homework_count} 次作业相关消息",
                "confidence": 0.6
            })

    return sorted(candidates, key=lambda x: -x["confidence"])
```

### 8.3 触发策略

```python
class HomeworkSyncStrategy:
    """
    1. 定时触发：每天 7:00 和 17:00 自动执行
    2. 手动触发：用户点击"立即同步"按钮
    3. 登录触发：检测到微信登录时自动执行
    """

    def should_remind_sync(self, last_pc_login_time):
        """判断是否需要提醒用户同步手机消息"""
        days_offline = (datetime.now() - last_pc_login_time).days
        if days_offline > 1:
            return True, f"您有 {days_offline} 天未登录，建议同步手机消息"
        return False, None

    async def scheduled_sync(self):
        """定时同步任务"""
        while True:
            now = datetime.now()
            target_times = ["07:00", "17:00"]
            current_time = now.strftime("%H:%M")

            if current_time in target_times:
                await self.sync_homework()

            await asyncio.sleep(60)  # 每分钟检查一次
```
