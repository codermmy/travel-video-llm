# 项目系统文档

`my-spec/system/` 只保留当前系统说明、模块文档和知识库，不再承载 spec 执行流程。

## 推荐阅读

- `project/01-overview.md`
- `project/02-architecture.md`
- `project/03-module-catalog.md`
- `frontend/INDEX.md`
- `backend/INDEX.md`
- `knowledge/INDEX.md`

## 使用方式

1. 先看 `project/` 了解当前产品口径和主链路。
2. 进入具体改动前，到 `frontend/` 或 `backend/` 找模块入口。
3. 遇到环境坑、框架坑、稳定排障结论，再写进 `knowledge/`。
