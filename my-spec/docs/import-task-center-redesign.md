# 导入任务中心重设计

## 目标

- 把“导入任务”从历史流水页重构为状态中心。
- 用户进入页面后，第一眼就能知道当前是否有任务在跑、是否有失败需要处理。
- 导入动作继续留在别的入口发起，导入中心只负责回看状态和历史。

## 当前问题

- 页面把说明文案、统计、筛选、历史任务、失败恢复动作都放在同一层，重点不明确。
- 每个任务默认展开四段阶段卡片，信息密度过高，历史记录很难快速扫读。
- 页面本身没有直接导入入口，失败任务的恢复动作也依赖跳转到其他页面。
- 当前任务、失败任务、已完成历史没有做优先级分层。

## 新的信息结构

### 1. 导入中心主页

- 主页只保留三类信息：
  - 当前最重要的一件事
  - 需要处理的任务
  - 最近完成的记录
- 首页不再承载阶段明细，不做页内展开。
- 首页不再提供 `导入照片` 动作。
- 当存在多批次同时导入时，首页顶部改为批次总览，不只突出某一个任务。

### 2. 任务详情页

- 点击任意任务记录后进入详情页。
- 详情页承载：
  - 完整阶段时间线
  - 关键计数
- 详情页不再承担“继续导入”“回到导入中心”这类额外动作。

## 卡片策略

- 首页顶部使用单张总览卡片：
  - 单个进行中任务时，卡片本身可点击进入详情
  - 多个进行中任务时，卡片只展示并行概况，不再绑定到单个任务
- 首页列表统一改为轻量行卡片，只显示：
  - 来源
  - 状态
  - 时间
  - 一行摘要
- 所有任务卡片都直接点击进入详情，不保留单独“查看详情”按钮。
- 复杂阶段信息全部下沉到详情页，避免首页视觉负担。

## 功能边界

- 本次只重构前端页面结构和交互，不改导入任务数据模型。
- 不新增后端接口。
- 不处理自动导入逻辑。
- 允许新增页面并调整既有入口跳转。

## 实施任务

- 重构导入中心主页，删除页内重明细布局和冗余导入动作。
- 新增任务详情页。
- 把失败弹层和顶部任务横幅改为直达详情页。
- 明确支持多批次并行导入的展示方式。
