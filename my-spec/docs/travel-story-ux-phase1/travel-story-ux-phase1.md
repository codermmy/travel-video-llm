# 需求文档：旅行故事体验打磨 Phase 1

- 文档状态：Implementing
- 优先级：P0
- 负责人：AI 协作
- 创建时间：2026-04-02
- 最后更新：2026-04-02

## 1. 背景

- 当前事件详情页会直接展示大段全文故事，用户点进来先面对阅读负担，而不是浏览照片。
- 当前幻灯片使用卡片式字幕，视觉语言更像播放器浮层，和照片内容、项目暖色旅程基调不一致。
- 当前产品主任务已经明确为“自动聚合照片后进行浏览与播放”，故事需要辅助照片，而不是压过照片。

## 2. 目标

- 把事件详情页重构为“照片片段浏览器”，让用户按照片组浏览旅程。
- 把幻灯片改成“电影字幕 + 轻镜头运动”的播放体验。
- 在不改后端主分组规则的前提下，快速提升成品感和观感。

## 3. 非目标

- 本期不做视频导出。
- 本期不做复杂 3D 转场、粒子、重特效。
- 本期不改后端 chapter / photoGroup 分组算法。
- 本期不做音乐节拍卡点。

## 4. 用户场景

- 用户进入事件详情页，希望先浏览这次旅程的片段照片，再决定是否展开读完整故事。
- 用户播放幻灯片时，希望像看一段轻量 vlog / 电影回忆片，而不是看普通图片轮播。

## 5. 需求拆解

### 5.1 功能需求

- 事件详情页以 `chapter` 作为主浏览单元，每段展示对应照片和轻文案。
- 全文故事下沉为折叠阅读区，不再首屏直出。
- 幻灯片改为 scene 播放模型，支持章节序幕、照片页、章节尾声。
- 照片页字幕只使用轻量 lower-third，不再展示卡片式字幕。

### 5.2 交互 / 界面要求

- Hero 保留封面、地点、时间、心情与主要操作。
- Hero 下展示一段简短故事引子。
- 每个 chapter 卡片展示代表图拼贴、章节标题、轻文案和照片数，点击后展开该段照片。
- 幻灯片照片页加入轻微 Ken Burns 式缩放/平移。
- 章节引言和总结改成独立转场页，不再压在照片上。

### 5.3 数据 / 状态要求

- 前端直接复用现有 `chapters` 和 `photos` 数据，不新增 API。
- chapter 通过 `photoStartIndex/photoEndIndex` 切片得到对应照片。
- 幻灯片内部新增 scene 派生状态，不落库。

## 7. 方案约束

- 保持当前事件详情 API 契约不变。
- 优先复用现有 Expo / React Native 能力，不引入重型依赖。
- 视觉风格需要回归项目已有暖色旅程体系。

## 8. 风险与待确认

- chapter 当前是按约 10 张切片，不是严格 5-10 张自适应分组；本期接受该现实。
- 轻镜头运动需要控制幅度，避免裁掉照片主体。
