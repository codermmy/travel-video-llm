# 单设备隐私优先需求拆分索引

> 状态：可用于并行开发分派
>
> 最后更新：2026-03-31
>
> 关联主文档：`single-device-privacy-first-prd.md`

---

## 1. 目标

将“单设备、默认不上图、端侧生成、显式增强入口”的总 PRD 拆成可并行推进的多个需求包，便于后续交给多个 AI agent 独立实现。

拆分原则：

- 每个需求包都要有独立的产品目标
- 每个需求包都要有明确 owner 边界
- 需求之间允许存在依赖，但应尽量减少代码写入冲突
- 技术契约优先统一，避免多个 agent 在 schema 和数据流上各自发挥

### 00. 总契约

- 文档：`single-device-workstream-03-android-ondevice-vision.md`

- 文档：`single-device-workstream-04-default-story-pipeline.md`

### 05. 事件详情、封面、幻灯片与本地导出

- 文档：`single-device-workstream-05-local-event-view-and-slideshow.md`

- 可并行性：高

## 3. 推荐并行顺序

### 第 0 批：先统一契约

1. `00-总契约`

### 第 1 批：可先并行

1. `01-单设备壳层与用户中心`
2. `03-Android 端侧识别 MVP`
3. `05-事件详情、封面、幻灯片与本地导出`

### 第 2 批：依赖契约与端侧结果

1. `02-本地导入与本地媒体模型`
2. `04-默认故事生成链路重构`

### 第 3 批：在默认路径稳定后接入

1. `06-可选云端增强入口`

---

## 4. 关键依赖关系

- `00` 是全部需求包的前置输入
- `03` 为 `04` 提供端侧识别输出 schema 和移动端实现方向
- `02` 为 `05` 提供本地媒体引用模型
- `04` 为 `05` 提供默认故事与章节文本来源
- `06` 独立于默认故事链路，但需要复用 `00` 的数据契约与 `05` 的事件详情入口

---

## 5. 当前仓库中的主要改造压力点

以下旧实现与新产品方向冲突，后续需求包会分别接住：

- 导入链路仍生成并上传缩略图
- 后端默认故事生成仍依赖公开图片 URL
- 事件详情、相册浏览、幻灯片仍优先使用服务端图片 URL
- 根布局和 Profile 页仍保留登录、同步、邮箱账号叙述
- 项目总览文档仍以“多设备同步”作为核心价值之一

---

## 6. 使用建议

如果后续要让多个 AI agent 并行开发，建议每个 agent 至少拿到：

1. 主 PRD：`single-device-privacy-first-prd.md`
2. 本索引文档
3. 对应的单个 workstream 文档
4. 该 workstream 涉及的现有代码文件列表

不要直接把总 PRD 丢给所有 agent 并要求“各自理解后实现”，那样高概率会在 schema、路由和数据来源上分叉。
