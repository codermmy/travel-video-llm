## 3. 页面边界

- 页面 01：无数据时的唯一启动页
- 页面 02：有数据时的主首页
- 页面 14：首页回忆流卡片长按 Sheet

## 4. 主要文件边界

- 允许修改：
  - `mobile/app/(tabs)/events.tsx`
  - `mobile/src/components/timeline/TimelineEventCard.tsx`
  - 与首页直接相关的轻量子组件
- 谨慎修改：
  - `mobile/app/(tabs)/_layout.tsx`
  - `mobile/src/components/ui/revamp.tsx`
- 不应修改：
  - 地图页面
  - 事件详情页
  - 播放器页面

## 5. 实现要求

- 冷启动欢迎态要求：
  - 只能有一个主 CTA：`整理最近 200 张`
  - `手动补导入` 必须保留，但明确降级为次入口
  - 隐私承诺必须是短卡片，不允许回退成长文案堆叠
- 首页要求：
  - 顶部标题 `回忆`
  - 轻量状态胶囊 `整理中 N`
  - 最近回忆 Hero 卡
  - 状态中心入口
  - 回忆流筛选
  - 按月份组织的回忆流
- 事件卡要求：
  - 媒体优先
  - 大图封面
  - 精简元信息
  - 长按必须有可见快捷操作 Sheet

## 6. 非目标

- 不改地图探索逻辑。
- 不改事件详情内部布局。
- 不改播放器本体。
- 不新增新的导入能力，只重构入口层级和展现方式。

## 8. 验收标准

- 无数据时，首页只呈现欢迎态，不再出现多个竞争空态卡片。
- 有数据时，首屏无需滚动即可同时看到最近回忆 Hero 和当前整理状态。
- 首页不再以地图或导入弹窗心智作为第一视图。
- 长按事件卡后，快捷操作不是隐藏逻辑，而是有明确 UI。

## 9. 验证方式

- `cd mobile && npm run lint`
- `cd mobile && npm run typecheck`
- 手动核对：
  - 无数据状态
  - 有数据状态
  - 整理中状态
  - 待更新状态
  - 事件卡长按状态

- 只有页面 01、02、14 都能在 HTML 中找到一一对应布局，并与实现逐项核对通过后，才允许勾总 PRD 的 `A1-A4`。
