Canvas Alignment Demo

我当前理解的是：我们不是播放器，而是在预览一支已经定规格的视频。

这个页面只对齐单张照片镜头的处理方式。重点不是“页面 UI”，而是“成片长什么样”：16:9 和 9:16 都先是完整视频画布，然后照片尽可能占满大部分画面；不匹配比例时，只留下干净黑边；字幕永远贴在整张视频画布底部的固定电影字幕线附近。

1. 先有视频画布 先确定整支视频是 16:9 还是 9:16，再讨论照片怎么放进去。
2. 照片尽量占满 同方向时优先尽量占满；但凡比例不一致，仍先保证完整显示，不强裁。
3. 黑边是标准画布 不做模糊铺底，不做补背景；凡是比例不一致，都直接用纯黑留白兜底。
4. 字幕贴着画面 字幕跟随视频画布的固定底部字幕线，不随着横图竖图的尺寸变化而收窄。
这份 demo 在刻意验证什么

我这次除了“横版视频中的横图 / 横版视频中的竖图 / 竖版视频中的竖图 / 竖版视频中的横图”，还补了“同方向但比例不一致”的场景。你主要看四件事： 照片占比是不是足够大，比例不一致时是不是优先完整显示，黑边是不是足够干净，字幕是不是始终跟着整张视频的固定底部字幕线。

字幕气质 像已经烧录进成片里的电影字幕，不像播放器黑条，也不像短视频控件。
字体方向 改成暖白无衬线，小字重、低存在感，更稳更清楚，不和照片争主角。
可读性处理 只保留轻阴影和极弱底部托底，增强可读性，但不制造明显的 UI 条形感。
横竖屏规则 横版更低，竖版略高，但仍是一套系统。字幕永远按视频画布，而不是按照片框对齐。
A. 16:9 视频 + 横图
横图直接全屏

这是最理想情况。横图进入横版成片，照片应该直接占满整个 16:9 视频画布，不留边，不加假背景。

照片全屏，字幕跟着整张横版视频的底部安全线出现。
B. 16:9 视频 + 4:3 横图
同方向，但不裁图

这是你刚补充的关键场景。虽然照片也是横图，但它不是 16:9，而是更接近 4:3。这里应该优先完整显示，所以让它高度贴满视频高度，左右留下黑边。

4:3 Landscape Photo
视频固定字幕线
画面完整保留，不为了填满 16:9 去牺牲原始构图。
统一策略不是“横图就一定铺满宽度”，而是“先完整显示，再由黑边兜底”。
C. 16:9 视频 + 竖图
竖图满高，两侧黑框

你刚才强调的核心场景。竖图在横版成片里不做模糊铺底，直接让照片高度贴满视频高度，左右保留黑边。

人站在山口，风景不需要被填满，留白本身就是电影画布。
即使中间是竖图，字幕仍横跨整张横版视频底部，不缩到竖图中间那一条里。
D. 9:16 视频 + 竖图
竖图几乎全屏

竖版成片遇到竖图时，同样应该尽可能让照片成为整个画面的主体，而不是缩成一张中间卡片。

竖版视频里，字幕依然固定在整张竖版视频底部的统一位置。
E. 9:16 视频 + 3:4 竖图
同方向，但不裁图

竖版成片里也要遵守同一条规则。即使照片也是竖图，但如果它更接近 3:4，而不是 9:16，就优先完整显示，让宽度贴满，顶部和底部保留黑边。

3:4 Portrait Photo
视频固定字幕线
即使都是竖图，也不是一律铺满；比例不对就保持完整，让黑边来承担适配。
竖版视频中的 3:4 竖图，应该是上下黑边，而不是强行拉伸或裁切成 9:16。
F. 9:16 视频 + 横图
横图满宽，上下黑框

反向场景同理。横图进入竖版成片时，不裁成奇怪构图，不做模糊铺满，上下直接保留黑边。

视频固定字幕线
公路被夕阳拉得很长，镜头不需要假装它原本就是竖着的。
即使横图只占中间区域，字幕也不跟着横图跑，仍落在整张竖版视频底部。
如果你认可这个方向，我下一步会据此继续细化两件事： 1. 横版和竖版的视频字幕线分别定多高更合适。 2. 序幕、章节、蒙太奇是否也要遵守同一条视觉安全线。 另外这版已经明确了一个统一适配规则：默认使用 `contain`，优先完整显示照片，不做强裁切。
