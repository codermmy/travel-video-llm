# PRD - events-timeline-view

## 1. 背景与目标

- **背景**：当前"事件"Tab 使用卡片列表展示事件，缺乏时间维度的连续感。用户希望以时间线的方式浏览旅行事件，更直观地感受时间流动和回忆脉络，类似 QQ 空间的说说/日志时间线体验。

- **业务目标**：
  - 提升事件浏览的沉浸感和时间感知
  - 让用户更容易按时间维度回顾旅行记忆
  - 优化视觉体验，提升产品质感

- **本次成功标准**：
  - 事件列表成功改造为时间线视图
  - 按月分组展示，月份标题包含统计信息
  - 用户可正常点击进入事件详情
  - 视觉效果美观，交互流畅

## 2. 范围定义

### In Scope

- 改造 `mobile/app/(tabs)/events.tsx` 的列表布局为时间线视图
- 新增时间线组件（垂直轴线 + 节点）
- 新增月份分组标题组件（含统计信息）
- 改造事件卡片样式（适配时间线布局）
- 缩略图预览（3张横排）

### Out of Scope

- 不新增 Tab，复用现有"事件"Tab
- 不修改事件详情页
- 不修改后端 API（复用现有 `listEvents` 接口）
- 不修改数据模型
- 不做年份分组（仅按月）
- 不做时间线动画效果（首版）

## 3. 用户场景

### 角色

- 普通用户：浏览自己的旅行事件

### 关键路径

1. 用户进入"事件"Tab
2. 看到按月分组的时间线视图
3. 月份标题显示"2026年2月 · 3个事件 · 45张照片"
4. 每个事件显示：标题、日期、地点、3张缩略图
5. 用户点击某个事件卡片
6. 跳转到事件详情页

### 异常路径

| 场景 | 处理方式 |
|------|----------|
| 无事件 | 显示现有空状态（保持不变） |
| 某月无事件 | 跳过该月，不显示 |
| 事件无照片 | 缩略图区域显示占位图标 |
| 事件无地点 | 显示"地点待补充" |
| 加载失败 | 显示现有错误状态（保持不变） |
| 分页加载更多 | 追加到对应月份分组，或新增月份分组 |

## 5. 非功能要求

- **性能**：
  - 首屏渲染时间 < 500ms（与现有持平）
  - 滚动流畅，无明显卡顿（60fps）
  - 分组计算在前端完成，不增加 API 调用

- **稳定性**：
  - 大量事件（100+）时分组逻辑正常
  - 跨年事件正确分组

- **安全性**：
  - 无新增安全风险（纯前端 UI 改造）

- **兼容性**：
  - iOS 和 Android 表现一致
  - 适配不同屏幕宽度

## 6. 风险与待确认

### 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| SectionList 性能 | 大量分组时可能卡顿 | 使用 `getItemLayout` 优化，必要时虚拟化 |
| 月份计算逻辑 | 跨时区可能导致分组错误 | 统一使用本地时区处理 |

### 待确认问题

- 无（需求已在对话中澄清完毕）

- 后续可考虑：时间线动画、年份折叠、搜索/筛选等增强功能
